Gunakan struktur role berikut sebagai hirarki resmi sistem (dari paling tinggi → paling rendah):

1. Superadmin

2. Director

3. GM

4. Manager HR

5. Manager

6. Team Lead

7. Staff

Admin (khusus opsional akses tools)

Kamu harus gunakan role tersebut untuk approval routing.

Berikut approval workflow yang WAJIB kamu implementasikan:

1. Promote to Manager
Step 1 → Direct Manager
Step 2 → HR Manager
Step 3 → Director

2. Terminate Employee
Step 1 → Division Manager
Step 2 → HR Manager

3. Transfer Division
Step 1 → Current Manager
Step 2 → Target Division Manager
Step 3 → HR Manager

Setiap workflow harus menghasilkan array approval seperti:

[
  { level: 1, approver_role: "Manager" },
  { level: 2, approver_role: "Manager HR" },
  { level: 3, approver_role: "Director" }
]


Generate approval otomatis saat request dibuat.
Simpan logika ini sebagai aturan tetap.
