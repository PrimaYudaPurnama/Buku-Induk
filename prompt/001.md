Ini konteks inti sistem:

Sistem punya entitas utama: User, Role, Division, AccountRequest, Approval, EmployeeHistory, Document, file_metaData, Notification.

Sistem mendukung workflow HR enterprise dengan multi-level approval, document storage Cloudinary, employee lifecycle, dan notifikasi.

Ingat aturan fundamental:

* AccountRequest adalah entry point untuk permintaan akun baru.

* Approval adalah per-step approval multi-level berdasarkan role hierarchy.

* Jika Approval final disetujui → update status utama + buat EmployeeHistory + buat document (jika ada) + kirim notifikasi.

* Jika satu saja approval ditolak → seluruh request digugurkan.

Pahami seluruh model dan relasi:
AccountRequest → Approval → EmployeeHistory → Document → Notification

Simpan pemahaman ini sebagai konteks permanen di seluruh perintah saya selanjutnya.
